## 🎲 ギミック・謎解きの整理法

### 謎解きテンプレート

```
;;;見出し3+太字
暗号の謎
;;;

;;;枠線
発見場所：書斎の机の引き出し
外見：古い羊皮紙に不可解な文字列
;;;

;;;太字
暗号文
;;;
「WKUHH GRRU VKDOO EHDU WKH IXOO PRRQ」

;;;折りたたみ
ヒント1：アイデア50%
;;;
これはシーザー暗号（文字をずらす暗号）のようだ。

;;;折りたたみ
ヒント2：アイデア80%または知識80%
;;;
アルファベットを3文字分ずらしている。
A→D、B→E、C→F...という具合。

;;;ネタバレ
解答（GMのみ）
;;;
正解：「THREE DOOR SHALL BEAR THE FULL MOON」
意味：「3つの扉が満月を担う」
→ 満月の夜に3つの扉を特定の順序で開ける必要がある。
```

## 📋 情報整理のテクニック

### 重要度別情報管理

```
;;;見出し2
情報収集フェーズ
;;;

;;;ハイライト color=#e6ffe6
【必須情報】探索者が必ず得るべき情報
;;;
- 被害者は全員が同じ悪夢を見ていた
- 悪夢の内容は「海底都市の呼び声」
- 失踪事件は満月の夜に集中している

;;;ハイライト color=#fff3cd
【重要情報】シナリオ進行に役立つ情報
;;;
- 古い日記に邪教団の記述がある
- 教団は月の満ち欠けに合わせて儀式を行う
- 次の満月は3日後（時間制限）

;;;ハイライト color=#ffe6e6
【秘匿情報】条件付きで開示される情報
;;;
- 依頼人の妻は実は生存している
- 邪教団のリーダーは町の有力者
- 儀式を止める方法は古文書に記載

;;;折りたたみ
各情報の入手方法（GMのみ）
;;;
**必須情報の入手方法：**
- 警察署での聞き込み（必ず成功）
- 被害者の家族への取材

**重要情報の入手方法：**
- 図書館での調査（図書館50%）
- 古書店での情報購入

**秘匿情報の入手方法：**
- 邪教団員の尋問（心理学80%）
- 隠し部屋の発見（隠れる30%）
```

## 🎪 クライマックスシーンの構築

### 戦闘バランステンプレート

```
;;;見出し2+太字
最終決戦：儀式の阻止
;;;

;;;枠線+ハイライト color=#fff3cd
シーン概要：
邪教団が地下神殿で最終儀式を執行中。
探索者は儀式を阻止し、人質を救出する必要がある。
制限時間：10ラウンド
;;;

;;;見出し3
戦力バランス
;;;

;;;枠線
探索者側（想定）：
- 人数：3〜4人
- 平均HP：12
- 主要技能：射撃40%、格闘50%、回避50%
;;;

;;;枠線
敵側：
- 教団リーダー：1体（強敵）
- 一般信者：探索者数+1体
- 召喚クリーチャー：1体（ラウンド5で登場）
;;;

;;;ハイライト color=#ffe6e6
バランス調整ルール
;;;
**探索者が優勢な場合：**
- 信者を1〜2体追加
- リーダーの技能を+10%上昇

**探索者が劣勢な場合：**
- 信者のHPを-2
- 人質が脱出を手伝う（回避+20%）
```

## 🎨 配布用HTMLの最適化

### 印刷・配布設定のコツ

#### 1. ファイル名の付け方

```
良い例：
- CoC6th_古館の秘密_v1.2.html
- シナリオ_海底からの呼び声_完成版.html

悪い例：
- scenario.html（内容が分からない）
- 新しいファイル.html（検索しにくい）
```

#### 2. メタ情報の設定

config.yamlで以下を設定：

```yaml
metadata:
  title: "CoC6th シナリオ：古館の秘密"
  author: "あなたの名前"
  description: "推奨人数3-4人、プレイ時間4-6時間"
  version: "v1.2"
```

#### 3. 印刷用最適化

```
;;;見出し1
印刷時の注意事項
;;;

;;;枠線+ハイライト color=#fff3cd
このシナリオは以下の設定で印刷することを推奨します：
- 用紙サイズ：A4
- 印刷の向き：縦
- マージン：標準
- 背景色：印刷しない（チェックを外す）
;;;
```

## 💡 作業効率化のコツ

### 1. テンプレートファイルの活用

シナリオ作成用のテンプレートファイルを作成して再利用：

```
template_scenario.txt：
;;;見出し1
シナリオタイトル（要変更）
;;;

;;;枠線
基本情報：
- 推奨人数：
- 想定時間：
- 推奨技能：
- 舞台：
;;;

（以下、構成テンプレート）
```

### 2. バージョン管理

```
scenario_v1.0.txt  （初稿）
scenario_v1.1.txt  （初回テストプレイ後修正）  
scenario_v1.2.txt  （2回目テストプレイ後修正）
scenario_final.txt （配布版）
```

### 3. 分割ファイル作成

大きなシナリオは分割して管理：

```
01_introduction.txt    （導入部分）
02_investigation.txt   （情報収集）
03_climax.txt         （クライマックス）
04_appendix.txt       （付録・NPC）
```
